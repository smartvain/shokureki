# 手動テストケース

動作確認用のテストケース一覧。上から順に実施してください。

---

## 前提条件

- ローカル dev サーバーが起動している (`pnpm dev`)
- `.env.local` に全環境変数が設定済み
- DB マイグレーション済み (`pnpm drizzle-kit push`)
- GitHub OAuth ログイン済み

---

## 1. 設定（GitHub連携）

| # | 操作 | 期待結果 |
|---|------|----------|
| 1.1 | サイドバー「設定」をクリック | 設定ページが表示される |
| 1.2 | GitHub Personal Access Token を入力して「保存」 | 「GitHubトークンを保存しました」と表示 |
| 1.3 | 「接続テスト」をクリック | 「接続成功: {ユーザー名}」と表示 |
| 1.4 | リポジトリ一覧を確認 | 自分のリポジトリが表示される |
| 1.5 | 1つ以上のリポジトリを「選択」 | ボタンが「選択済み」に変わる |

## 2. ダッシュボード（活動収集）

| # | 操作 | 期待結果 |
|---|------|----------|
| 2.1 | ダッシュボードで「今日の活動を収集」をクリック | 「収集中・AI分析中...」表示後、収集結果が表示される |
| 2.2 | 実績候補が表示されることを確認 | タイトル・説明・カテゴリ・技術スタックが表示 |
| 2.3 | 候補の「承認」をクリック | 候補がリストから消え、Stats の確定済み実績が増加 |
| 2.4 | 別の候補で「スキップ」をクリック | 候補がリストから消える |
| 2.5 | 手動メモのみで収集（GitHub活動なしの日を想定） | テキストエリアにメモを入力 → 収集 → AI が実績候補を生成 |

## 7. 書類生成（職務経歴書）

> **前提:** テストケース 2（実績承認）、5（プロフィール）、6（職務経歴）を先に実施してデータを登録しておくこと。

### 7.1 生成ウィザード

| # | 操作 | 期待結果 |
|---|------|----------|
| 7.1.1 | サイドバー「書類一覧」→「新規作成」 | ウィザード Step 1 が表示される |
| 7.1.2 | フォーマット「逆編年体」を選択 →「次へ」 | Step 2（実績選択）に遷移 |
| 7.1.3 | 実績にチェック（1つ以上）→「次へ」 | Step 3（AI生成設定）に遷移 |
| 7.1.4 | 応募先企業・ポジション（任意）を入力 →「生成」 | ローディング → AI がコンテンツ生成 → Step 4 に自動遷移 |
| 7.1.5 | プレビューを確認 | 職務要約・スキル・職務経歴・自己PR が表示される |
| 7.1.6 | 「次へ」→ Step 5 で「下書き保存」 | 書類一覧ページにリダイレクト |

### 7.2 書類一覧・詳細

| # | 操作 | 期待結果 |
|---|------|----------|
| 7.2.1 | 書類一覧に作成した書類が表示される | タイトル・フォーマット・ステータス「下書き」が表示 |
| 7.2.2 | 「閲覧」をクリック | 書類詳細ページが表示される |
| 7.2.3 | 「確定」ボタンをクリック | ステータスが「確定」に変わる |
| 7.2.4 | 「PDF出力」をクリック | PDF ファイルがダウンロードされる |
| 7.2.5 | ダウンロードした PDF を開く | 日本語が正しく表示、レイアウトが崩れていない |
| 7.2.6 | 書類一覧に戻り「削除」→ 確認 → 削除 | 書類が消える |
